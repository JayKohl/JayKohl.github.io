Game of Life

Gameplay Image

Conway's Game of Life is a computer simulation known as a cellular automaton, designed to model complex systems using simple rules on a grid of cells.

This was my first large project at Full Sail University, a solo endeavor in which I fully developed the application. During Project and Portfolio One, I manipulated arrays and data structures within a Windows application to create Conway's Game of Life. This project helped me develop a strong foundation in data manipulation, user interaction, and game design principles.

Game Features

Render Conway's Game of Life in a .NET Application

The game renders a grid that represents individual cells, which can be toggled on and off by clicking on them with the mouse. Once a group of cells is activated, the game will simulate life and death according to the following rules:

Rule 1: Living cells with fewer than two living neighbors will die in the next generation.

Rule 2: Living cells with more than three living neighbors will die in the next generation.

Rule 3: Living cells with two or three living neighbors will survive to the next generation.

Rule 4: Dead cells with exactly three living neighbors will come to life in the next generation.

Tool Strip Buttons

The game starts running by clicking the **Start** button and can be paused using the **Pause** button. If the game is paused, it can advance by one generation using the **Next** button found on the tool strip.

Randomizing the Universe

The universe can be randomly populated either by using the current time or a seed variable. The seed is set via a dialog box, and randomization is activated through a menu selection.

Emptying the Universe

The universe is cleared of all living cells when the **New** or **Clear** menu item is selected, allowing for a fresh start or a new configuration.

Saving and Importing

The current state and size of the universe can be saved in **PlainText** file format. Users can choose the save file name via a save file dialog box. Additionally, previously saved **PlainText** files can be loaded into the current universe, with the universe resizing to match the size of the file being read.

Universe Information

The current generation and number of living cells are tracked and displayed on the status strip at the bottom of the window application, providing real-time information about the simulation's progress.

Adjustable Options

The number of milliseconds between new generations can be adjusted through a dialog box. The width and height of the current universe can also be customized via a modal dialog box, allowing users to tailor the simulation to their preferences.

Cell Information

The neighbor count for each individual cell is rendered on the grid and can be toggled in the **View** menu, offering insight into each cell's interactions.

View Menu

Features such as neighbor count, grid lines, and heads-up display (HUD) can all be toggled on or off in the **View Menu**, allowing users to customize their view of the simulation.

Importing Patterns

PlainText life patterns can be downloaded from the **Life Lexicon** website and imported into the current universe. The universe size remains unchanged when importing new patterns, and importing will not clear the universe before reading in the file.

Game Colors

Users can select individual colors for the grid, background, and living cells through a modal dialog box, enabling a personalized visual experience.

Universe Boundary

Users can choose how the game treats the edges of the universe, with options to switch between toroidal (wrap-around) and finite boundaries.

Context-Sensitive Menu

A context menu strip allows users to change various options in the application, providing a convenient way to adjust settings without navigating through complex menus.

Heads-Up Display (HUD)

The HUD displays essential information such as cell count, boundary type, and universe size. This feature can be toggled on or off in the **View Menu**.

Settings

Changes to universe size, timer interval, and color options persist even after the program is closed and reopened. Users have additional menu items, **Reload** and **Reset**, to revert to the last saved settings or return to the application's default settings.
